Memory Allocation Algorithms

1. First Fit

Allocates the first hole that is big enough. Searching starts from the beginning of the set of holes. We can stop as soon as we find a free hole that is large enough.

✅ Advantages

  • Fastest — searches as little as possible.
  • Simple to implement.

❌ Disadvantages

  • Leaves unused holes (external fragmentation).
  • Concentrates allocation at one end of memory.

2. Best Fit

Allocates the smallest hole that is big enough. Must search the entire list. Produces the smallest leftover hole.

✅ Advantages

  • Generally better memory utilization.
  • Minimizes wasted space in the short term.

❌ Disadvantages

  • Slower — must search entire list.
  • Creates many tiny unusable holes.

3. Worst Fit

Allocates the largest hole. Produces the largest leftover hole, which may be more useful than tiny holes from Best Fit.

✅ Advantages

  • Largest leftover hole is more likely usable.
  • Reduces production of small unusable holes.

❌ Disadvantages

  • Slow — requires searching entire list.
  • Breaks up large free blocks.

4. Next Fit

Like First Fit, but starts searching from the last allocated block. Wraps around to the beginning if it reaches the end.

✅ Advantages

  • Very fast allocation.
  • Distributes allocations more evenly.

❌ Disadvantages

  • May break up the largest free block more often.
Page Replacement Algorithms

1. FIFO (First In, First Out)

The oldest page in memory is replaced first. Uses a simple queue — the page that has been in memory the longest is evicted when a new page needs to be loaded.

✅ Advantages

  • Simplest to implement — just a queue.
  • Low overhead with minimal bookkeeping.

❌ Disadvantages

  • Suffers from Bélády's anomaly — more frames can cause more faults.
  • Ignores page usage patterns.

2. LRU (Least Recently Used)

Replaces the page that has not been used for the longest time. Based on the idea that pages used recently are likely to be used again soon (temporal locality).

✅ Advantages

  • Good performance — exploits temporal locality.
  • Does not suffer from Bélády's anomaly.

❌ Disadvantages

  • Expensive to implement — needs timestamping or stack.
  • High overhead for every memory access.

3. Optimal (OPT / MIN)

Replaces the page that will not be used for the longest time in the future. This is the theoretically best algorithm — it produces the fewest possible page faults.

✅ Advantages

  • Produces the minimum number of page faults.
  • Serves as a benchmark for other algorithms.

❌ Disadvantages

  • Impossible to implement in practice — requires future knowledge.
  • Used only for comparison and analysis.
